Generate the offline-first infrastructure: 1) app/views/service_worker/service_worker.js.erb — Service Worker that caches the app shell (HTML, CSS, JS, fonts) on install, intercepts fetch requests with a cache-first strategy for assets and network-first for API calls, falls back to cached versions when offline. 2) public/dexie.min.js — note this will be a placeholder; the actual Dexie.js library should be vendored. 3) app/javascript/controllers/offline_controller.js — Stimulus controller that monitors navigator.onLine, shows online/offline status indicator, triggers sync when coming back online. 4) app/controllers/translations_controller.rb — serves locale JSON at /translations/:locale.json by reading and merging the YAML locale files into a flat JSON structure for offline i18n. 5) Add the service worker route and translations route to config/routes.rb. The service worker should be served from the root path /service_worker.js for maximum scope.
